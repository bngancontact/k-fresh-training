# TC003 - Verify Updated Account Information Is Displayed Correctly After Successful Update

| Field | Details |
|---|---|
| **ID** | TC003 |
| **SCREEN** | Edit Account Information — `https://ecommerce-playground.lambdatest.io/index.php?route=account/edit` |
| **TEST CASE** | Verify updated account information is displayed correctly after a successful account update |
| **DESCRIPTION** | Verify that after the user updates account information successfully and returns to the **My Account** page, the updated values are retained and displayed correctly when the user opens the **Edit Account** page again. |

---

## PRE-CONDITION

- The user has a valid registered account.
- The user is logged in successfully.
- The user has already completed the account update successfully.
- The success message **"Success: Your account has been successfully updated."** was displayed on the **My Account** page.

---

## TEST DATA

| Field | Value |
|---|---|
| Expected First Name | `Ngan_Update` |
| Expected Last Name | `Bao_Update` |
| Expected Telephone | `0987654321` |

---

## TEST STEPS

| Step | Action |
|---|---|
| 1 | From the **My Account** page, verify that the success message for account update is displayed. |
| 2 | Click the **Edit Account** link or **Edit your account information** shortcut again. |
| 3 | Observe the values displayed in the **First Name**, **Last Name**, and **Telephone** fields. |
| 4 | Compare the displayed values with the updated values entered in the previous test case. |

---

## EXPECTED RESULTS

- The user can open the **Edit Account** page successfully.
- The **First Name** field displays `Ngan_Update`.
- The **Last Name** field displays `Bao_Update`.
- The **Telephone** field displays `0987654321`.
- The updated account information is retained correctly after the successful update.
- No old value is displayed in the updated fields.
- No error message is shown.
